# SimpleSettleApp — iOS port (built & shipped via GitHub Actions)

iOS equivalent of the Android `SimpleSettleApp`: a thin `WKWebView` wrapper around
`https://simplesettleapp.com` that reproduces every behaviour of the Android
`MainActivity`. The Xcode project is generated by **XcodeGen** and built/signed/
uploaded to **TestFlight** by **Fastlane** running on a **GitHub Actions** macOS
runner — so you never touch a Mac yourself.

## Files


## Feature mapping (Android → iOS)
| Android (`MainActivity.kt`) | iOS |
|---|---|
| `WebView` + settings | `WKWebView` (`WebViewController.swift`) |
| `enableEdgeToEdge` + insets | safe-area handling (`ContentView.swift`) |
| `AndroidInterface` JS bridge | `WKScriptMessageHandler` + injected JS shim |
| `onGoogleLoginSuccess(idToken)` | `evaluateJavaScript` |
| Play Services Auth | GoogleSignIn-iOS SDK |
| `SwipeRefreshLayout` | `UIRefreshControl` |
| progress bar | `UIProgressView` + `estimatedProgress` KVO |
| UA `SimpleSettleAndroid` | UA `SimpleSettleiOS` |
| back button | edge-swipe back (`allowsBackForwardNavigationGestures`) |
| `tel:`/`mailto:` external | `decidePolicyFor` + `UIApplication.open` |

# What YOU must do once (the parts tied to your identity)

These need your Apple/Google accounts, so I can't do them for you. Everything else
is automated. Budget ~1 hour.

### 1. Apple Developer Program
- Enroll: https://developer.apple.com/programs/ ($99/year). This is the iOS analog
  of your Google Play account + keystore.
- Note your **Team ID** (Developer portal → Membership) → secret `DEVELOPMENT_TEAM`.

### 2. App Store Connect API key (lets CI log in without 2FA)
- App Store Connect → Users and Access → **Integrations / Keys** → generate an
  **App Store Connect API key** (Admin or App Manager role).
- You get: **Key ID**, **Issuer ID**, and a one-time `.p8` file download.
- Secrets:
  - `ASC_KEY_ID` = Key ID
  - `ASC_ISSUER_ID` = Issuer ID
  - `ASC_KEY_CONTENT` = base64 of the `.p8` file
    (`base64 -i AuthKey_XXXX.p8` on mac/linux, or any base64 tool)

### 3. Create the App Store Connect app record
- App Store Connect → Apps → **+** → New App. Bundle ID `com.simplesettleapp.com`.
  Name: "Simple Settle App".

### 4. Google iOS OAuth client
- Google Cloud Console (same project as the existing web client `466145634768-…`)
  → Credentials → Create OAuth client → **iOS**, with your Bundle ID.
- Put the values into `SimpleSettleApp/Info.plist`:
  - `GIDClientID` = the new iOS client ID
  - URL scheme = its **reversed** form (`com.googleusercontent.apps.<id>`)
  - `GIDServerClientID` is already the web client ID — leave it.

### 5. A private "certs" repo for Fastlane match
- Create an empty **private** GitHub repo, e.g. `simplesettle-certs`.
- Secrets:
  - `MATCH_GIT_URL` = its URL (`https://github.com/<you>/simplesettle-certs.git`)
  - `MATCH_PASSWORD` = any passphrase you choose (encrypts the stored certs)
  - `MATCH_GIT_BASIC_AUTHORIZATION` = base64 of `<github-username>:<a PAT with repo scope>`
    (so CI can read/write the certs repo)

### 6. Push this project to GitHub and add the secrets
- This folder is **not a git repo yet**. Initialize and push to a GitHub repo.
- In that repo: Settings → Secrets and variables → Actions → add all secrets above:
  `DEVELOPMENT_TEAM`, `ASC_KEY_ID`, `ASC_ISSUER_ID`, `ASC_KEY_CONTENT`,
  `ASC_TEAM_ID` (optional), `MATCH_GIT_URL`, `MATCH_PASSWORD`,
  `MATCH_GIT_BASIC_AUTHORIZATION`.

### 7. First run
- Either push to `main`, or open the **Actions** tab → "iOS Build & TestFlight" →
  **Run workflow**. The first run creates the signing certificate + profile (stored
  in your certs repo), builds the app, and uploads it to TestFlight.
- After processing, the build appears in TestFlight for testing. When ready,
  submit for App Store review from App Store Connect (this last step is a manual
  Apple review form).

## App icon
`SimpleSettleApp/Assets.xcassets/AppIcon.appiconset/icon-1024.png` — the purple
wallet icon, 1024×1024, **no alpha, square, full-bleed** (App Store compliant). It
was processed from the supplied artwork: the transparent rounded corners were
removed and the background rebuilt as a uniform purple so iOS can apply its own
corner mask cleanly. To replace it, just drop a new opaque 1024×1024 PNG (no alpha,
no pre-rounded corners) at that path with the same filename.

## Bundle ID note
The app uses `com.simplesettleapp.com` (matching the Google iOS OAuth client and the
Android `applicationId`). It is referenced in `project.yml`, `.github/workflows/ios.yml`
(`APP_IDENTIFIER`), `fastlane/Appfile`, and `fastlane/Matchfile`. If you ever change
it, update all four — and re-register it on the Google iOS OAuth client too.

## Backend note (important — same as before)
Android's ID token has audience = the **web** client ID; the iOS token's audience =
the **iOS** client ID. Make your backend accept **both** client IDs as valid
audiences when verifying the Google ID token, or iOS logins will be rejected.
